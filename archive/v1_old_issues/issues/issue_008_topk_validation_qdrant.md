## Summary

Qdrant does not properly validate the limit (top_k) parameter in search operations, or provides unclear error messages when validation fails.

## Reproduction Steps

1. Create a collection with data
2. Execute search with invalid limit values (0, -1)
3. Observe behavior and error messages

## Expected Result

Invalid limit values should be rejected with clear error messages. System should remain stable.

## Actual Result

Invalid values may be accepted or cause unclear errors. Error messages may not specify valid range.

## Root Cause

Missing or insufficient input validation for limit parameter. Error messages may not include valid range. May not check for zero or negative values.

## Impact

- Invalid input may cause unexpected behavior
- Poor error messages hinder debugging
- System may behave unpredictably

## Workaround

- Validate limit on client side before sending to Qdrant
- Use reasonable default values
- Add client-side error handling

## Proposed Fix

- Add strict input validation with checks for <=0 and upper limits
- Improve error messages with valid range
- Document valid limit range
- Add validation tests
