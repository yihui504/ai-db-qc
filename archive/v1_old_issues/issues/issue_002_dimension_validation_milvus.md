## Summary

Milvus incorrectly rejects dimension value of 1, which should be a valid vector dimension. The error message returned is empty, providing no useful information to the developer.

## Reproduction Steps

1. Attempt to create a collection with vector dimension = 1
2. Use valid schema configuration
3. Execute the creation operation

## Expected Result

Collection creation should succeed with dimension = 1, or if not supported, a clear error message should indicate the valid dimension range.

## Actual Result

Collection creation is rejected with empty error message: ''. No indication of valid dimension range or reason for rejection.

## Root Cause

The validation logic for dimensions has an incorrect lower bound check. It appears to be using dimension > 1 instead of dimension >= 1, or there's a hardcoded minimum dimension that's too restrictive. Error handling code doesn't provide meaningful messages.

## Impact

- Developers cannot use 1D vectors which are valid in many use cases
- Empty error messages make troubleshooting difficult
- Unclear what the actual valid dimension range is

## Workaround

- Pad vectors to a higher dimension (e.g., add zeros)
- Use a different vector database if 1D vectors are required
- Check source code or experiment to find actual dimension limits

## Proposed Fix

- Update dimension validation to accept dimension >= 1
- Or document the actual valid dimension range clearly
- Ensure error messages always provide useful information (e.g., 'Dimension must be between 1 and 32768')
- Add unit tests for boundary values (1, 2, 32768, 32769)
