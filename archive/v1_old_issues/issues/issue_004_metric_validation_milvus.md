## Summary

Milvus accepts invalid or unsupported metric types when creating indexes without proper validation. This can lead to runtime errors or unexpected behavior.

## Reproduction Steps

1. Create a collection with standard vector schema
2. Insert data into the collection
3. Attempt to create an index with an invalid metric type
4. Observe whether the operation is rejected

## Expected Result

Index creation should be rejected with clear error message. Only documented and supported metric types should be accepted.

## Actual Result

Invalid metric type is accepted without validation error. May cause runtime errors or incorrect search results later.

## Root Cause

The index creation code does not validate the metric type parameter against a whitelist of supported metrics. Validation may be missing entirely, only checking string format, or checking at a different layer.

## Impact

- Invalid configurations lead to unpredictable behavior
- No immediate feedback on incorrect configuration
- Failures occur later in the pipeline making them harder to diagnose

## Workaround

- Carefully read Milvus documentation for supported metrics
- Use only well-documented metric types (L2, IP, COSINE)
- Validate metric types on client side before sending to Milvus

## Proposed Fix

- Implement strict metric type validation at index creation
- Document all supported metrics clearly
- Add metric type validation tests
- Provide better error messages with list of supported metrics
- Consider auto-detecting appropriate metric based on vector type
