## Summary

Qdrant has issues with dimension validation, either rejecting valid dimension values or providing unclear/incomplete error messages when validation fails.

## Reproduction Steps

1. Attempt to create collections with various dimension values
2. Test boundary values (1, 2, high values)
3. Observe error messages and behavior

## Expected Result

Valid dimensions should be accepted. Invalid dimensions should be rejected with clear error messages specifying the valid range.

## Actual Result

Some valid dimensions may be rejected. Error messages may be unclear or incomplete. Actual valid range not clearly communicated.

## Root Cause

Dimension validation logic may have incorrect bounds. Error messages may not include valid range information. Validation may be inconsistent across different API endpoints.

## Impact

- Developers struggle with configuration
- Poor error messages hinder troubleshooting
- Unclear what values are actually supported

## Workaround

- Experiment with different dimension values
- Use common dimensions (e.g., 128, 256, 512, 768, 1024)
- Check Qdrant documentation for examples

## Proposed Fix

- Update dimension validation to correct bounds
- Improve error messages with valid range
- Document valid dimension range clearly
- Add validation tests for boundary values
