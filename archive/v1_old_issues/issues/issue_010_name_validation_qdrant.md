## Summary

Qdrant accepts collection names that should be reserved or invalid, including system names, names with invalid characters, or names that could cause conflicts.

## Reproduction Steps

1. Attempt to create collections with various names
2. Test reserved names, special characters, edge cases
3. Observe validation behavior

## Expected Result

Reserved names should be rejected. Names with invalid characters should be rejected. Clear error messages with naming rules.

## Actual Result

Some invalid names may be accepted. Validation may be insufficient. Error messages may be unclear.

## Root Cause

No list of reserved system names. May only check basic validity (length, characters). Does not prevent naming conflicts.

## Impact

- May conflict with system collections
- Potential for confusion or privilege issues
- Unclear naming rules

## Workaround

- Use a naming convention (e.g., prefix with app name)
- Avoid obviously reserved names
- Check existing collections before creating

## Proposed Fix

- Implement strict name validation with reserved names list and character rules
- Document naming rules clearly
- Add validation tests
- Provide clear error messages
