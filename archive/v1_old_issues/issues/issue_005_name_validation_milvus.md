## Summary

Milvus accepts collection names that should be reserved or invalid, including reserved system names like 'system', which can lead to naming conflicts and potential security issues.

## Reproduction Steps

1. Attempt to create a collection with a reserved name (e.g., 'system')
2. Observe whether the operation is rejected
3. Attempt operations on the reserved name collection

## Expected Result

Collection creation should be rejected for reserved names with error like 'Collection name 'system' is reserved and cannot be used'. Clear list of reserved names should be provided.

## Actual Result

Reserved names are accepted without validation error. Potential for conflicts with system operations.

## Root Cause

The collection name validation logic does not maintain a list of reserved system names, may only check for basic validity (length, characters) but not semantic validity, does not check for potential conflicts with internal system collections.

## Impact

- User collections may conflict with system collections
- Potential for privilege escalation or confusion
- Difficult to distinguish user vs system collections
- Unclear which names are allowed

## Workaround

- Use a naming convention for user collections (e.g., prefix with company name)
- Manually avoid obviously reserved names
- Check existing collections before creating new ones

## Proposed Fix

- Implement strict collection name validation with reserved names list:
  - system, default, admin, root
  - Names starting with _ or .
  - Names with special characters
- Document all reserved names and naming rules
- Add validation tests for edge cases
- Provide clear error messages with examples of valid names
