## Summary

Qdrant fails when operating on large datasets (100k+ vectors). Operations exhibit unacceptable performance degradation or fail entirely.

## Reproduction Steps

1. Create a collection optimized for large datasets
2. Insert a large dataset (100k+ vectors)
3. Perform various operations (batch inserts, searches, filtering, updates, deletes)
4. Monitor performance and errors

## Expected Result

All operations should complete successfully. Performance should scale reasonably with dataset size. No crashes or timeouts. Acceptable response times for common operations.

## Actual Result

Operations fail or timeout on large datasets. Performance degrades disproportionately. May require breaking data into smaller collections. Not suitable for large-scale production use.

## Root Cause

- Indexing performance issues on large datasets
- Inefficient data structures or algorithms
- Memory management issues
- Insufficient optimization for large-scale operations
- Lack of proper batching or pagination
- I/O bottleneck with large data files

## Impact

- Cannot handle large-scale production workloads
- Limited to small/medium datasets
- Poor performance on large data
- May drive users to other solutions

## Workaround

- Partition data across multiple collections
- Use sharding in distributed mode
- Implement client-side caching
- Optimize queries and filters
- Use appropriate index types for large datasets

## Proposed Fix

- Optimize indexing algorithms for large datasets
- Implement efficient batching and streaming
- Add proper memory management and garbage collection
- Improve query optimization for large collections
- Add partitioning or sharding support
- Document recommended collection sizes and scaling strategies
